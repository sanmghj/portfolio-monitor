# Portfolio Monitor

FastAPI + React single-server portfolio monitor for local use.

The backend serves CSV-based API data, and the React app in `ui/` is built into static files and served by FastAPI at `/`.

## Overview

- Local personal server
- No database
- CSV-only storage
- React UI served by FastAPI
- User entry flow based on `name + birth_date`
- Portfolio types: `dividend`, `general`
- Portfolio CRUD
- Holding CRUD
- Holding insights, news, events, and simple price history
- Prompt CRUD for AI workflow notes

## Python Setup

```powershell
$venv = 'D:\dev\.venv_port'
& "$venv\Scripts\python.exe" -m pip install --upgrade pip
& "$venv\Scripts\python.exe" -m pip install -r requirements.txt
Copy-Item .env.example .env -ErrorAction SilentlyContinue
```

## Frontend Setup

If you use `nvm` on Windows:

```powershell
nvm install 22.14.0
nvm use 22.14.0
node -v
npm -v
```

Install UI dependencies:

```powershell
Set-Location ui

Set-Location ..
```

## Production-Style Local Run

Build the React UI first:

```powershell
Set-Location ui
npm run build
Set-Location ..
```

Run the FastAPI server:

```powershell
$env:PATH = "D:\dev\.venv_port\Scripts;$env:PATH"
uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Open:

- App: `http://127.0.0.1:8000/`
- Swagger docs: `http://127.0.0.1:8000/docs`
- Legacy `/main` routes now redirect to `/`

## Development Workflow

Frontend dev server:

```powershell
Set-Location ui
npm run dev
```

Backend dev server:

```powershell
Set-Location D:\dev\portfolio-monitor
$env:PATH = "D:\dev\.venv_port\Scripts;$env:PATH"
uvicorn app.main:app --reload
```

Notes:

- `ui/vite.config.ts` proxies `/api` to `http://127.0.0.1:8000` during frontend development.
- For the actual one-server flow, always run `npm run build` and then start FastAPI.

## Seed Data

The repository includes starter CSV files:

- `data/users.csv`
- `data/portfolios.csv`
- `data/holdings.csv`
- `data/latest_prices.csv`
- `data/earnings_events.csv`
- `data/news_headlines.csv`
- `data/prompts.csv` is created automatically when needed

If these files are missing, the app recreates them and seeds a starter user plus starter portfolios.

## API Endpoints

Users:

- `GET /api/v1/users`
- `POST /api/v1/users`
- `POST /api/v1/users/enter`

Portfolios:

- `GET /api/v1/users/{user_id}/portfolios`
- `GET /api/v1/users/{user_id}/portfolios/{portfolio_id}`
- `POST /api/v1/users/{user_id}/portfolios`
- `PUT /api/v1/users/{user_id}/portfolios/{portfolio_id}`
- `DELETE /api/v1/users/{user_id}/portfolios/{portfolio_id}`

Holdings:

- `GET /api/v1/users/{user_id}/holdings`
- `POST /api/v1/users/{user_id}/holdings`
- `PUT /api/v1/users/{user_id}/holdings/{holding_id}`
- `DELETE /api/v1/users/{user_id}/holdings/{holding_id}`
- `GET /api/v1/users/{user_id}/holdings/{holding_id}/insights`
- `GET /api/v1/users/{user_id}/holdings/{holding_id}/price-history`
- `POST /api/v1/users/{user_id}/holdings/{holding_id}/refresh`

Prompts:

- `GET /api/v1/users/{user_id}/prompts`
- `POST /api/v1/users/{user_id}/prompts`
- `PUT /api/v1/users/{user_id}/prompts/{prompt_id}`
- `DELETE /api/v1/users/{user_id}/prompts/{prompt_id}`

Dashboard / refresh:

- `GET /api/v1/users/{user_id}/dashboard/summary`
- `POST /api/v1/refresh/prices`
- `POST /api/v1/refresh/events`
- `POST /api/v1/refresh/news`
